## Directory Structure

Bifrost follows Next.js App Router conventions with a clear separation of concerns.

```text
apps/bifrost/
├── src/
│   ├── app/              # Next.js App Router pages
│   ├── components/       # React components
│   ├── constants/        # Schemas and constants
│   ├── hooks/            # Custom React hooks
│   ├── lib/              # Core logic and stores
│   ├── providers/        # Context providers
│   ├── utils/            # Utility functions
│   └── assets/           # Static assets
├── public/               # Public static files
└── package.json
```

## App Directory

The `src/app` directory uses Next.js 16 App Router with file-based routing.

### Admin Pages (Protected)

```text
app/(admin-pages)/
├── companies/
│   ├── [slug]/          # Company detail page
│   ├── create-company/  # Create company form
│   └── page.tsx         # Companies list
├── organization/
│   └── page.tsx         # Organization settings
├── students/
│   ├── [slug]/          # Student profile
│   └── page.tsx         # Students list
└── layout.tsx           # Admin layout wrapper
```

<Note>
  The `(admin-pages)` route group allows pages to share a common layout without affecting the URL structure.
</Note>

## Core Directories

### Constants

```text
constants/
└── schemas/    # Zod validation schemas
```

Contains Zod schemas for form validation and data validation across the application.

### Hooks

```text
hooks/          # Custom React hooks
```

Custom hooks for shared logic, data fetching, and state management.

### Lib

```text
lib/
└── stores/     # Zustand state stores
```

Core application logic and Zustand stores for global state management.

### Providers

```text
providers/      # React context providers
```

Context providers for Convex, Clerk, theming, and other global concerns.

### Utils

```text
utils/          # Utility functions
```

Helper functions, formatters, and shared utilities.

### Assets

```text
assets/
└── navet/      # Navet-specific assets
```

Static assets including images, logos, and other media files.

## Workspace Packages

Bifrost depends on shared workspace packages:

```json
"@workspace/backend"       // Convex backend functions
"@workspace/auth"          // Authentication utilities
"@workspace/shared"        // Shared types and utilities
"@workspace/ui"            // Shared UI components
"@workspace/typescript-config"  // Shared TypeScript config
```text

<Note>
  These packages are part of the Yggdrasil monorepo and enable code sharing across applications.
</Note>

## Build Configuration

### Scripts

```bash
pnpm dev        # Start development server (port 3001, with Turbopack)
pnpm build      # Production build
pnpm start      # Start production server
pnpm lint       # Run Next.js linting
pnpm check      # Run Biome linter/formatter
pnpm check:fix  # Auto-fix Biome issues
```

### Development Tools

- **Turbopack**: Enabled for faster development builds
- **Biome**: Code linting and formatting
- **TypeScript**: Type checking
- **Sentry**: Error tracking and monitoring
- **PostHog**: Analytics and feature flags
