# Project Structure

Hugin follows a standard Next.js App Router structure with organized component and utility directories.

## Directory Overview

```
hugin/
├── src/
│   ├── app/                    # Next.js App Router pages
│   ├── components/             # React components
│   ├── lib/                    # Core logic and schemas
│   ├── providers/              # Context providers
│   └── assets/                 # Static assets
├── public/                     # Public static files
└── config files               # Configuration files
```

#### Root Layout

The root layout (`src/app/layout.tsx:30`) wraps the entire app with:

1. **ClerkProvider** - Authentication context
2. **ConvexClientProvider** - Convex backend connection
3. **ThemeProvider** - Dark/light mode support
4. **PostHogProvider** - Analytics tracking
5. **Header** - Navigation component
6. **Toaster** - Toast notification system

```tsx
<html lang="nb" suppressHydrationWarning>
  <body>
    <PostHogProvider>
      <ClerkProvider>
        <ConvexClientProvider>
          <ThemeProvider>
            <Header />
            {children}
            <Toaster />
          </ThemeProvider>
        </ConvexClientProvider>
      </ClerkProvider>
    </PostHogProvider>
  </body>
</html>
```

#### Input Card Components

All input cards follow a consistent pattern:

- Accept a TanStack Form field API
- Display label with optional required indicator
- Handle validation and error display
- Support readonly mode for viewing responses
- Wrapped in a Card component for consistent styling

<Tip>
  The input card components are exported as a barrel module from `src/components/input-cards/index.ts:1` for cleaner imports.
</Tip>

#### Event Feedback Schema

Defines the structure and validation rules for feedback forms using Zod:

```typescript
// src/lib/schema/event-feedback-schema.ts:3
export const eventResponseFromSchema = z.object({
  satisfaction: z.int().min(1, "Må velge et alternativ").max(5),
  impression: z.int().min(1, "Må velge et alternativ").max(5),
  expectation: z.int().min(1, "Må velge et alternativ").max(5),
  toughts: z.string().min(1, "Venligst fyll inn").max(1000),
  improvements: z.string().min(1, "Venligst fyll inn").max(1000),
  want_to_work: z.enum(["ja", "nei"]),
  word_of_mouth: z.array(z.string()).min(1),
  other: z.string().max(1000),
});
```

## Configuration Files

### Next.js Configuration

**File:** `next.config.ts:1`

Key configurations:

- **Transpiled packages:** `@workspace/ui` for shared components
- **Image domains:** Convex CDN, Clerk, and local development
- **PostHog rewrites:** Privacy-focused analytics proxy at `/relay-aXgZ/*`
- **Sentry integration:** Error tracking with source maps
- **Turbopack:** Enabled for faster development builds

```typescript
const nextConfig: NextConfig = {
  transpilePackages: ["@workspace/ui"],
  cacheComponents: true,
  images: {
    remotePatterns: [
      { hostname: convexSite },
      { hostname: "img.clerk.com" },
      { hostname: "127.0.0.1", port: "3210" },
    ],
  },
  async rewrites() {
    return [
      {
        source: "/relay-aXgZ/:path*",
        destination: "https://eu.i.posthog.com/:path*",
      },
    ];
  },
};
```

## Routing Structure

Hugin uses file-based routing with the following structure:

| Route | File | Purpose |
|-------|------|----------|
| `/` | `app/page.tsx` | Redirect to ifinavet.no |
| `/event-feedback/[slug]` | `app/event-feedback/[slug]/page.tsx` | Event feedback form |
| `/event-feedback/[slug]/response` | `app/event-feedback/[slug]/response/page.tsx` | View submitted response |

<Info>
  All routes under `/event-feedback/*` require authentication via Clerk.
</Info>
