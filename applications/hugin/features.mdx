# Hugin Features

Hugin provides a comprehensive event feedback collection system with user-friendly interfaces and robust validation.

## Event Feedback Forms

The primary feature of Hugin is collecting structured feedback from event attendees.

### Form Schema Validation

All feedback forms use Zod schema validation defined in `src/lib/schema/event-feedback-schema.ts:3`:

```typescript
export const eventResponseFromSchema = z.object({
  satisfaction: z.int().min(1).max(5),
  impression: z.int().min(1).max(5),
  expectation: z.int().min(1).max(5),
  toughts: z.string().min(1).max(1000),
  improvements: z.string().min(1).max(1000),
  want_to_work: z.enum(["ja", "nei"]),
  word_of_mouth: z.array(z.string()).min(1),
  other: z.string().max(1000),
});
```

## Access Control & Validation

Hugin implements several layers of access control to ensure feedback integrity.

### Authentication

<Card title="Clerk Integration" icon="lock">
  All feedback routes require Clerk authentication. Unauthenticated users are redirected to sign-in.
  
  Implemented in `src/app/event-feedback/[slug]/page.tsx:20`
</Card>

### Attendance Verification

Before allowing feedback submission, Hugin verifies:

1. **Event has a feedback form** - The event must have an associated form
2. **User attended the event** - Checks against event registration records
3. **User hasn't already submitted** - Prevents duplicate responses

```typescript
const ableToAnswer = await fetchQuery(
  api.forms.checkIfCurrentUserAttendedTheEventAndShouldBeAbleToSubmit,
  { eventId: event._id },
  { token }
);
```

### Duplicate Prevention

If a user has already submitted feedback, they're shown a confirmation message with a link to view their existing response instead of the form.

## Response Management

### Form Submission

Feedback is submitted to the Convex backend via the `submitFormResponse` mutation:

```typescript
formResponseMutation({
  formId: event.formId!,
  data: {
    userId,
    eventId: event._id,
    ...value,
  },
});
```

### View Previous Responses

Users can view their previously submitted feedback at `/event-feedback/[slug]/response`:

- Read-only display of their answers
- Same visual format as the submission form
- Accessible only to the user who submitted the response
