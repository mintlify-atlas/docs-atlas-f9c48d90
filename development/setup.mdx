## Prerequisites

Before you begin, ensure you have the following installed on your system:

<CardGroup cols={2}>
  <Card title="Node.js" icon="node-js">
    Version 20 or higher is required
  </Card>
  <Card title="pnpm" icon="box">
    Version 10.22.0 (specified in package.json)
  </Card>
</CardGroup>

<Note>
  Yggdrasil uses **pnpm** as its package manager. The project specifies version 10.22.0, which will be automatically used if you have Corepack enabled.
</Note>

## Quick Start

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/yggdrasil.git
    cd yggdrasil
    ```
  </Step>

  <Step title="Install dependencies">
    Install all dependencies across the monorepo:
    
    ```bash
    pnpm install
    ```
    
    This will install dependencies for all apps and packages in the workspace.
  </Step>

  <Step title="Set up environment variables">
    Create `.env` files in the root and for each app you want to run. See the [Environment Variables](/development/environment) page for details.
    
    <Warning>
      The project requires several environment variables for Clerk, Convex, and Resend to function properly. Ensure all required variables are set before running the development server.
    </Warning>
  </Step>

  <Step title="Start development server">
    Run the development server for all apps:
    
    ```bash
    pnpm dev
    ```
    
    This will start:
    - **Midgard** on `http://localhost:3000`
    - **Bifrost** on `http://localhost:3001`
    - **Hugin** on `http://localhost:3003`
  </Step>
</Steps>

## Project Structure

The Yggdrasil monorepo is organized using Turborepo:

```
🌳 yggdrasil/
├── 📁 apps/
│   ├── 🌈 bifrost/      # Admin dashboard (port 3001)
│   ├── 🌍 midgard/      # Main website ifinavet.no (port 3000)
│   └── 🦅 hugin/        # Additional service (port 3003)
├── 📦 packages/
│   ├── 🗄️ backend/         # Convex backend logic
│   ├── 📧 emails/          # React email components
│   ├── 🔧 typescript-config/ # Shared TypeScript configuration
│   └── 🎨 ui/           # Shared React component library
├── 📁 documentation/      # Documentation files
├── 📜 package.json      # Monorepo-level scripts and dependencies
└── 📜 turbo.json        # Turborepo configuration
```

## Development Workflow

### Running Individual Apps

You don't always need to run all services. Use Turborepo's filtering to run specific apps:

<CodeGroup>
```bash Midgard Only
pnpm --filter midgard dev
```

```bash Bifrost Only
pnpm --filter bifrost dev
```

```bash Hugin Only
pnpm --filter hugin dev
```
</CodeGroup>

### Building the Project

To build all apps and packages:

```bash
pnpm build
```

<Note>
  The build process uses Turborepo's caching mechanism. Subsequent builds will be faster as unchanged packages are cached.
</Note>

### Code Quality Checks

The project uses Biome for linting and formatting:

<CodeGroup>
```bash Check All
pnpm check:all
```

```bash Auto-fix All
pnpm check:all:fix
```

```bash Check Specific App
pnpm --filter midgard check
```

```bash Fix Specific App
pnpm --filter midgard check:fix
```
</CodeGroup>

## Common Setup Issues

### Node Version Mismatch

If you encounter errors related to Node.js version:

```bash
# Check your Node version
node --version

# Should be 20.x or higher
```

<Note>
  Consider using a Node version manager like `nvm` or `fnm` to manage multiple Node.js versions.
</Note>

### pnpm Not Found

If `pnpm` is not installed:

```bash
# Enable Corepack (recommended)
corepack enable

# Or install pnpm globally
npm install -g pnpm@10.22.0
```

### Port Already in Use

If a port is already occupied:

<CodeGroup>
```bash Custom Port
# Run on a different port
pnpm --filter midgard dev -- -p 3005
```

```bash Kill Process
# Find and kill the process using the port
lsof -ti:3000 | xargs kill -9
```
</CodeGroup>
