## Component Library

The `@workspace/ui` package provides a comprehensive set of accessible, themeable, and composable React components. All components are built with Radix UI primitives and styled with Tailwind CSS.

## Import Examples

### Basic Component Import

```tsx
import { Button } from "@workspace/ui/components/button";
import { Input } from "@workspace/ui/components/input";

function MyForm() {
  return (
    <div>
      <Input placeholder="Enter your name" />
      <Button>Submit</Button>
    </div>
  );
}
```

### Card with Subcomponents

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter
} from "@workspace/ui/components/card";
import { Button } from "@workspace/ui/components/button";

function ProfileCard() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>User Profile</CardTitle>
        <CardDescription>Manage your profile settings</CardDescription>
      </CardHeader>
      <CardContent>
        {/* Content goes here */}
      </CardContent>
      <CardFooter>
        <Button>Save Changes</Button>
      </CardFooter>
    </Card>
  );
}
```

### Form with Validation

```tsx
import { Form } from "@workspace/ui/components/form";
import { Field } from "@workspace/ui/components/field";
import { Input } from "@workspace/ui/components/input";
import { Button } from "@workspace/ui/components/button";
import { useForm } from "react-hook-form";
import { z } from "zod";
import { zodResolver } from "@hookform/resolvers/zod";

const schema = z.object({
  email: z.string().email(),
  password: z.string().min(8),
});

function LoginForm() {
  const form = useForm({
    resolver: zodResolver(schema),
  });

  return (
    <Form {...form}>
      <Field name="email" label="Email">
        <Input type="email" />
      </Field>
      <Field name="password" label="Password">
        <Input type="password" />
      </Field>
      <Button type="submit">Login</Button>
    </Form>
  );
}
```

## Component Features

### Variant System

Many components use `class-variance-authority` (CVA) for managing variants:

- **Button**: 6 variants (default, destructive, outline, secondary, ghost, link) and 6 sizes
- **Badge**: Multiple color variants for different statuses
- Type-safe variant props with TypeScript

### Composition Pattern

Components support the `asChild` prop for composition:

```tsx
import { Button } from "@workspace/ui/components/button";
import Link from "next/link";

<Button asChild>
  <Link href="/dashboard">Go to Dashboard</Link>
</Button>
```

### Utility Function

All components use the `cn` utility for class name merging:

```tsx
import { cn } from "@workspace/ui/lib/utils";

// Combines Tailwind classes and resolves conflicts
const className = cn("px-4 py-2", "px-6", "bg-blue-500");
// Result: "py-2 px-6 bg-blue-500"
```

## Contributing

When adding new components to the library:

1. Follow monorepo conventions for branching and commits
2. Ensure components are fully typed with TypeScript
3. Build with accessibility in mind (use Radix UI primitives)
4. Include proper focus management and keyboard navigation
5. Document component usage and props
6. Run `pnpm lint` before committing
